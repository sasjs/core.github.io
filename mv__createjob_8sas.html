Detailed Description

Code is passed in as one or more filerefs. 

%* Step 1 - compile macros ;
filename mc url "https://raw.githubusercontent.com/sasjs/core/main/all.sas";
%inc mc;

%* Step 2 - Create some SAS code and add it to a job;
filename ft15f001 temp;
parmcards4;
    data some_code;
      set sashelp.class;
    run;
;;;;
%mv_createjob(path=/Public/app/sasjstemp/jobs/myjobs,name=myjob)

Parameters
path=The full path (on SAS Drive) where the job will be created
name=The name of the job
desc=The description of the job
precode=Space separated list of filerefs, pointing to the code that needs to be attached to the beginning of the job
code=Fileref(s) of the actual code to be added
access_token_var=The global macro variable to contain the access token
grant_type=valid values are "password" or "authorization_code" (unquoted). The default is authorization_code.
replace=select NO to avoid replacing any existing job in that location
contextname=Choose a specific context on which to run the Job. Leave blank to use the default context. From Viya 3.5 it is possible to configure a shared context - see https://go.documentation.sas.com/?docsetId=calcontexts&docsetTarget=n1hjn8eobk5pyhn1wg3ja0drdl6h.htm&docsetVersion=3.5&locale=en

1 /**
2  @file
3  @brief Extract the log from a completed SAS Viya Job
4  @details Extracts log from a Viya job and writes it out to a fileref
5 
6  To query the job, you need the URI. Sample code for achieving this
7  is provided below.
8 
9  ## Example
10 
11  First, compile the macros:
12 
13  filename mc url
14  "https://raw.githubusercontent.com/sasjs/core/main/all.sas";
15  %inc mc;
16 
17  Next, create a job (in this case, a web service):
18 
19  filename ft15f001 temp;
20  parmcards4;
21  data ;
22  rand=ranuni(0)*1000;
23  do x=1 to rand;
24  y=rand*4;
25  output;
26  end;
27  run;
28  proc sort data=&syslast
29  by descending y;
30  run;
31  ;;;;
32  %mv_createwebservice(path=/Public/temp,name=demo)
33 
34  Execute it:
35 
36  %mv_jobexecute(path=/Public/temp
37  ,name=demo
38  ,outds=work.info
39  )
40 
41  Wait for it to finish, and grab the uri:
42 
43  data _null_;
44  set work.info;
45  if method='GET' and rel='self';
46  call symputx('uri',uri);
47  run;
48 
49  Finally, fetch the log:
50 
51  %mv_getjoblog(uri=&uri,outref=mylog)
52 
53  This macro is used by the mv_jobwaitfor.sas macro, which is generally a more
54  convenient way to wait for the job to finish before fetching the log.
55 
56 
57  @param [in] access_token_var= The global macro variable to contain the access
58  token
59  @param [in] mdebug= set to 1 to enable DEBUG messages
60  @param [in] grant_type= valid values:
61  @li password
62  @li authorization_code
63  @li detect - will check if access_token exists, if not will use sas_services
64  if a SASStudioV session else authorization_code. Default option.
65  @li sas_services - will use oauth_bearer=sas_services.
66  @param [in] uri= The uri of the running job for which to fetch the status,
67  in the format `/jobExecution/jobs/$UUID/state` (unquoted).
68  @param [out] outref= The output fileref to which to APPEND the log (is always
